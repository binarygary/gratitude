<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IndexedDB sync state -> user-visible UI | Local status/error fields become visible user guidance and recovery actions. |
| preserved conflict copy -> UI | Private local journal text may be shown in a conflict review area. |
| product copy -> user expectation | Help/Today copy sets expectations about backup and fresh-device restore. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | Information Disclosure | conflict review UI | mitigate | Show preserved local copy only inside the user's authenticated/local page; do not log it or expose it in global alerts/admin surfaces. |
| T-01-05-02 | Repudiation | status component | mitigate | Render text labels and `aria-live` status so users can see whether entries are local, pending, synced, failed, rejected, or conflict. |
| T-01-05-03 | Tampering | conflict discard action | mitigate | Require `confirm('Discard the saved local copy and keep the synced version?')` before clearing `conflict_local_payload`. |
| T-01-05-04 | Information Disclosure | misleading restore claims | mitigate | Replace across-device/full-restore copy with explicit backup and fresh-browser limitation copy from the UI spec. |
| T-01-05-05 | Denial of Service | sync retry action | mitigate | Disable retry while busy and use Plan 04 chunking/status rules; UI cannot trigger parallel duplicate retry loops. |
</threat_model>
