## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge interior gaps, inline separators |
| sm | 8px | Compact status rows, badge padding, table cell status spacing |
| md | 16px | Default component gap, form control spacing, alert padding |
| lg | 24px | Card body padding, save/status area padding, Help section padding |
| xl | 32px | Existing `main` vertical spacing between major cards |
| 2xl | 48px | Empty-state vertical breathing room inside History |
| 3xl | 64px | Reserved for page-level spacing only; do not use inside sync status components |

Exceptions: Interactive text buttons and menu items must keep a minimum 44px touch target on mobile. Existing app cards may keep `rounded-2xl`; new sync badges use `rounded-lg`, and new controls use the existing `rounded-xl` control pattern.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Rules: Phase 1 introduces no new font sizes and no new font weights beyond 400 and 600. Status text, validation messages, and Help copy use 14px. Entry textareas and body content use 16px. Sync panel headings use 18px. Existing page headings may remain as-is, but new Phase 1 UI must not add hero-scale type.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-base-100`; retro card/nav surfaces are white, dim body is `#353f52` | Page background, content surfaces, input interiors |
| Secondary (30%) | `border-base-300/50`, `text-base-content/70`, dim card `#3c485c` | Card borders, secondary copy, quiet table metadata, status panel boundaries |
| Accent (10%) | `--color-primary: oklch(62% 0.09 180)` | Primary Save entry button, focus rings, brand dot, syncing/pending emphasis only |
| Destructive | `text-error`, `alert-error`, `btn-error` from daisyUI theme | Rejected-entry validation failures and confirmed discard of a preserved local conflict copy only |

Accent reserved for: Save entry, Send magic link already in the shell, focus rings, syncing/pending status badges, and one inline "Retry sync" action. Do not use accent for every link, table row, card heading, or informational copy.

Status color mapping:

| Sync State | Label | Visual Treatment |
|------------|-------|------------------|
| `local` | Saved locally | Neutral badge: `badge badge-ghost`, `text-base-content/70` |
| `pending` | Sync pending | Primary-soft badge: `badge badge-primary badge-outline` |
| `syncing` | Syncing | Primary-soft badge with stable width; no layout-shifting spinner |
| `synced` | Synced | Success badge: `badge badge-success badge-outline` |
| `failed` | Sync failed | Warning/info alert plus `badge badge-warning badge-outline`; retry action visible |
| `rejected` | Needs edit | Error alert plus `badge badge-error badge-outline`; no automatic retry |
| `conflict` | Review needed | Accent/warning alert plus `badge badge-warning badge-outline`; recovery action visible |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Save entry |
| Empty state heading | No entries yet. |
| Empty state body | Save today's reflection to start your history. |
| Error state | This entry is saved on this device, but it could not sync. Check your connection and retry. |
| Destructive confirmation | Discard local conflict copy: "Discard the saved local copy and keep the synced version?" |

Required state copy:

| State | User-Facing Copy | Action |
|-------|------------------|--------|
| Saved locally | Saved on this device. | None |
| Sync pending | Waiting to sync when you are signed in and online. | None |
| Syncing | Syncing entry. | None |
| Synced | Synced backup is up to date. | None |
| Failed | This entry is saved on this device, but sync did not finish. | Retry sync |
| Rejected | This entry is saved on this device, but it needs changes before it can sync. | Edit entry |
| Conflict | A newer synced version exists. The synced version is shown, and your local copy is preserved for review. | Review local copy |

Restore-limit copy that must replace current overpromising language:

| Surface | Required Copy |
|---------|---------------|
| Today intro | Sign in to back up entries that sync successfully. Full restore on a new device is limited in this beta. |
| Login threshold alert | Sync is optional. It backs up entries that sync successfully; keep exporting if you need a full portable copy. |
| Help account section | Creating an account backs up accepted entries. A fresh browser may not show your full journal until restore is added and verified. |

## Interaction Contract

| Surface | Contract |
|---------|----------|
| Today save area | Keep the existing Save entry button as the primary action. Add a compact status line beside or below "Last saved" that always reflects the current entry state. On mobile, stack button, timestamp, and status vertically with 8px gaps. |
| Today actionable failures | Show failed, rejected, and conflict states as a single `alert` directly below the save row. Use `role="alert"` for rejected/conflict and `role="status" aria-live="polite"` for failed retry feedback. |
| History list | Add a status column or compact status cell per row. The status label must be visible without opening the entry. Search and sort must not hide failed/rejected/conflict entries unless the row itself is filtered out by the user's search. |
| Help page | Replace restore-overpromise copy in the account section. Do not add a marketing explainer or hero; keep the current short card structure. |
| Rejected entries | Do not show Retry sync until the user edits and saves the entry again. The user action is Edit entry, returning to the relevant day. |
| Failed entries | Retry sync is allowed. While retrying, keep the control width stable and disable duplicate clicks. |
| Conflict entries | Show server-canonical text as the primary entry content, preserve access to the local copy, and require confirmation before discarding the local conflict copy. |
| Request-level batch errors | Show one concise app-level alert in History or Today: "Sync could not send this batch. Try again in a moment." Item-level errors still belong on the affected entry rows. |

Accessibility requirements: every status badge must include text, not color alone. Status transitions after save or retry use `aria-live="polite"`. Validation and rejected states use `role="alert"`. Buttons must keep descriptive visible labels; do not introduce icon-only controls for sync recovery in Phase 1.
