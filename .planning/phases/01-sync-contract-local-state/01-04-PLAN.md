<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `/api/sync/push` response -> IndexedDB | Server results rewrite browser-local private entry data and status. |
| IndexedDB status -> retry behavior | Persisted state controls whether entries are sent again. |
| network failures -> user-visible state | Request failures become local errors shown later in UI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | Tampering | `applySyncResult` canonical writes | mitigate | Replace local fields only from accepted/skipped canonical server payloads and preserve losing local content for conflicts. |
| T-01-04-02 | Denial of Service | `pushUnsyncedEntries` retry loop | mitigate | Chunk at 50 entries, mark in-flight entries `pending`, and stop automatic retries for `rejected` and `conflict`. |
| T-01-04-03 | Information Disclosure | sync failure metadata | mitigate | Store generic request-failure string and structured validation field errors only; no console logging of prompt text. |
| T-01-04-04 | Repudiation | stale/corrupt local sync state | mitigate | Persist `server_payload`, `conflict_local_payload`, `sync_error`, and `last_sync_attempt_at` so outcomes can be explained and recovered. |
</threat_model>
