<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> `/api/sync/push` | Untrusted authenticated JSON can contain many entries, duplicate dates, stale timestamps, and private prompt text. |
| server canonical result -> IndexedDB | Browser will trust this response to rewrite local state in later plans. |
| validation errors -> browser | Rejection feedback must be actionable without exposing private text in logs or error details. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Tampering | `SyncController` duplicate handling | mitigate | Reject duplicate entry dates after validation so one batch cannot overwrite the same day twice. |
| T-01-02-02 | Tampering | server-newer/client-newer conflict semantics | mitigate | Test `skipped` server-newer and `upserted` client-newer behavior through the real HTTP route and `UpsertEntry`. |
| T-01-02-03 | Information Disclosure | sync errors/results | mitigate | Include canonical entries only for accepted/skipped items owned by the authenticated user; rejected item errors contain field names/messages, not submitted prompt text. |
| T-01-02-04 | Denial of Service | sync payloads | mitigate | Preserve Plan 01 `entries` max/list validation and per-field max validation while adding duplicate handling. |
</threat_model>
