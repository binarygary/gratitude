<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Laravel write routes | Authenticated browser JSON crosses into server validation for private journal content. |
| validation errors -> browser UI | Server validation details are returned to the browser and must not echo raw journal content beyond field names/messages. |
| sync payload size -> PHP/database | User-controlled batch size and prompt text can create resource pressure. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Denial of Service | `EntryPayloadRules` / `SyncController` | mitigate | Enforce `MAX_PROMPT_LENGTH = 5000`, `MAX_BATCH_ENTRIES = 50`, and `MAX_DEVICE_ID_LENGTH = 64` before upsert work. |
| T-01-01-02 | Tampering | `updated_at` validation | mitigate | Reject client timestamps more than 900000 ms in the future to bound clock-skew overwrites. |
| T-01-01-03 | Information Disclosure | validation errors | mitigate | Return Laravel field errors only; do not add logs or response fields that include submitted prompt text. |
| T-01-01-04 | Spoofing | `/entries/upsert`, `/api/sync/push` | accept | Existing routes remain behind `auth` middleware; broader session/CSRF posture is Phase 2 scope and is not expanded here. |
</threat_model>
