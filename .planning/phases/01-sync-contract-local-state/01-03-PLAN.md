<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| old IndexedDB schema -> new app code | Existing user data crosses a browser storage migration boundary. |
| sync status -> retry loop | Persisted local status determines whether private entries are resent to the server. |
| test harness -> browser storage API | Node tests simulate IndexedDB and must not mask destructive migration behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | Dexie version 2 migration | mitigate | Versioned upgrade preserves existing prompt fields and derives explicit status without clearing the table. |
| T-01-03-02 | Denial of Service | automatic retry selection | mitigate | Exclude `pending`, `synced`, `rejected`, and `conflict` from `listUnsyncedEntries()` so known bad states do not retry indefinitely. |
| T-01-03-03 | Information Disclosure | local sync errors | mitigate | Store structured validation messages or generic strings only; do not add console logging or UI diagnostics containing full journal text. |
| T-01-03-04 | Repudiation | stale/corrupt local state | mitigate | Persist `sync_status`, `sync_error`, `server_payload`, and `conflict_local_payload` so later UI can explain and recover states. |
</threat_model>
