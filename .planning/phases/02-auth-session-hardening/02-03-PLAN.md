<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| email link -> `/auth/magic-link/{token}` | Untrusted URL path and query parameters cross into signed URL middleware and token lookup. |
| token table -> session guard | A persisted one-time token authorizes a Laravel session only if valid, unused, and unexpired. |
| operator CLI -> token table | Cleanup command deletes rows from `magic_login_tokens`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Spoofing | `MagicLinkController::consume` | mitigate | Require hashed token match, unused `used_at`, unexpired `expires_at`, and existing user before login. |
| T-02-03-02 | Tampering | `signed:relative` route | mitigate | Keep Laravel signed middleware and render `InvalidSignatureException` as a recovery redirect. |
| T-02-03-03 | Repudiation | reused token handling | mitigate | Mark token `used_at` before login and reject subsequent uses with reused-link copy. |
| T-02-03-04 | Denial of Service | `magic_login_tokens` table | mitigate | Add `auth:prune-magic-links` to delete used and expired rows. |
| T-02-03-05 | Information Disclosure | invalid/reused link copy | mitigate | Recovery copy does not expose account existence or raw token details. |
</threat_model>
