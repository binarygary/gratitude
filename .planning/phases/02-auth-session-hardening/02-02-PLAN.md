<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> `/auth/magic-link/request` | Untrusted email and Turnstile token cross into auth request handling. |
| request attempts -> rate limiter cache | IP and normalized email values influence throttle buckets. |
| auth response -> unauthenticated user | Public response copy can leak account existence or abuse-control state if it differs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Denial of Service | `MagicLinkController::request` | mitigate | Run `TurnstileVerifier::verify()` before user/token/mail side effects. |
| T-02-02-02 | Denial of Service | `RateLimiter::for('magic-link-request')` | mitigate | Apply `5 per 15 minutes` by IP and `3 per hour` by normalized email. |
| T-02-02-03 | Information Disclosure | request responses | mitigate | Use exact status `If your email is valid, we sent a sign-in link.` for accepted, throttled, unknown, and failed-verification attempts where possible. |
| T-02-02-04 | Information Disclosure | throttle/verifier diagnostics | mitigate | Do not log private journal content, raw magic-link tokens, raw emails in user-facing responses, or Turnstile error codes. |
| T-02-02-05 | Tampering | `cf-turnstile-response` | mitigate | Treat the browser token as untrusted and accept it only after the server-side verifier returns successful. |
</threat_model>
