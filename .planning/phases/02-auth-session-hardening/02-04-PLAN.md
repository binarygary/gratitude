<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| magic-link request -> persisted token row | The user's remember-device choice persists until the link is consumed. |
| token row -> Laravel session guard | Token metadata controls remember-cookie issuance. |
| browser Axios -> session-authenticated write routes | Private entry writes cross from browser JavaScript to Laravel session routes. |
| cross-site request -> Laravel web middleware | Third-party sites may try to trigger authenticated writes using session cookies. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Spoofing | remember-device login | mitigate | Use `remember_device` stored on the validated magic-link token, default false, and configurable remember duration. |
| T-02-04-02 | Information Disclosure | session cookie configuration | mitigate | Preserve Laravel `web` guard/database sessions and document env keys in Plan 05; do not introduce JWT/Sanctum. |
| T-02-04-03 | Tampering | `/entries/upsert` | mitigate | Remove CSRF exemption and test missing-token rejection through `PreventRequestForgery`. |
| T-02-04-04 | Tampering | `/api/sync/push` | mitigate | Remove CSRF exemption and test missing-token rejection through `PreventRequestForgery`. |
| T-02-04-05 | Denial of Service | same-origin write behavior | mitigate | Add Axios `X-CSRF-TOKEN` header from existing Blade meta token so legitimate same-origin writes continue to work. |
</threat_model>
