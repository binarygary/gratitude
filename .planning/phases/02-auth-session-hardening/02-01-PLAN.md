<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Laravel auth request | Untrusted Turnstile token will later cross into the magic-link request controller. |
| Laravel -> Cloudflare Siteverify | Server sends only Turnstile token, secret, and remote IP to the third-party verification endpoint. |
| local/test bypass -> auth gate | Deterministic bypass must not become an accidental production allow-all path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Spoofing | `HttpTurnstileVerifier` | mitigate | Verify tokens server-side against Cloudflare Siteverify before Plan 02 sends mail. |
| T-02-01-02 | Information Disclosure | verifier error handling | mitigate | Do not log raw Turnstile tokens, secrets, emails, IP addresses, or Cloudflare error details. |
| T-02-01-03 | Elevation of Privilege | local/test bypass | mitigate | Bind bypass only when `services.turnstile.enabled` is false and environment is `local`/`testing`; non-local/test disabled or missing-secret states bind a fail-closed verifier so mail is not sent. |
| T-02-01-04 | Denial of Service | external Siteverify call | mitigate | Use Laravel HTTP timeout `TURNSTILE_TIMEOUT=3` and return unsuccessful on missing secrets, disabled non-local/test config, or failed HTTP responses. |
</threat_model>
