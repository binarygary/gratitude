# Checklists

## Definition of Done

- [ ] Tests added/updated and passing.
- [ ] Lint/format checks run for touched code.
- [ ] Docs/planning updates made for behavior changes.
- [ ] Deploy notes captured for risky changes.

## Release Checklist

- [ ] Pull latest changes and verify lockfiles are committed.
- [ ] Run tests and production build.
- [ ] Verify environment variables and secrets.
- [ ] Confirm queue/worker process expectations.
- [ ] Record release notes and rollback pointer.

## Dependency Update Checklist

- [ ] Review changelogs and breaking changes.
- [ ] Apply updates in a scoped PR.
- [ ] Run tests/build after updates.
- [ ] Document follow-up migration or runtime checks.
